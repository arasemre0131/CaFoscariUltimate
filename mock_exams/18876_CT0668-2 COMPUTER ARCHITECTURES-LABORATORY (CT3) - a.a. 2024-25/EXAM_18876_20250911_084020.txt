PART B: SHORT ANSWERS
1. Endianness refers to the order in which bytes are stored in memory for multi-byte data types. In little-endian systems, the least significant byte is stored at the lowest memory address, while in big-endian systems, the most significant byte is stored at the lowest address. This affects the interpretation and manipulation of multi-byte data types, such as integers and floating-point numbers.

2. The memory hierarchy in a computer system is designed to provide a balance between fast access, high capacity, and cost-effectiveness. Registers provide the fastest access but have the smallest capacity, while secondary storage (e.g., hard drives, solid-state drives) offer the largest capacity but slower access times. The intermediate levels, such as cache and main memory, aim to bridge the performance gap between the CPU and the slower storage. The trade-offs involve access speed, capacity, and cost per unit of storage.

3. Logical shifts and arithmetic shifts are two different types of bitwise operations in computer systems.
   - Logical shift: Shifts the bits of the operand to the left or right, filling the vacated positions with zeros. This effectively multiplies or divides the operand by a power of 2.
   - Arithmetic shift: Shifts the bits of the operand to the left or right, but preserves the sign bit (the most significant bit) by filling the vacated positions with copies of the sign bit. This is useful for signed integer operations, as it maintains the sign of the value.

4. The ARM instruction set architecture (ISA) is a RISC (Reduced Instruction Set Computer) design, in contrast to the traditional CISC (Complex Instruction Set Computer) architecture. The key differences are:
   - ARM ISA has a simpler and more efficient instruction set, with a focus on commonly used operations.
   - ARM processors have a streamlined pipeline and control unit, leading to faster instruction execution.
   - ARM's RISC design allows for reduced power consumption and better energy efficiency, making it well-suited for mobile and embedded applications.
   - The tradeoff is that RISC architectures may require more instructions to perform complex tasks, compared to CISC designs.

5. The BL (Branch with Link) instruction in ARM assembly is used to implement function calls. It performs the following tasks:
   - Stores the return address (the address of the next instruction after the BL) in the link register (R14 or LR).
   - Jumps to the specified target address, which is the entry point of the called function.
   - When the function returns, the return address stored in LR is used to jump back to the original caller, allowing the program to continue execution from the point of the function call.

2. The Von Neumann and Harvard architectures differ in their memory organization and instruction execution:

   Von Neumann architecture:
   - Uses a single memory for both instructions and data.
   - Instructions and data share the same bus, leading to potential bottlenecks during program execution.
   - Simpler memory management and control unit design.
   - Suitable for general-purpose computing tasks.

   Harvard architecture:
   - Employs separate memories for instructions and data.
   - Utilizes separate buses for instructions and data, allowing for concurrent access and higher throughput.
   - Requires more complex memory management and control unit design.
   - Well-suited for applications with high-performance requirements, such as digital signal processing (DSP) and real-time embedded systems.

   The key implications of these architectural differences are:
   - Von Neumann architecture may suffer from the "von Neumann bottleneck" due to the shared memory bus, limiting overall system performance.
   - Harvard architecture can achieve higher instruction and data throughput, as the separate buses allow for concurrent access.
   - Von Neumann is more suitable for general-purpose computing, while Harvard is advantageous for applications with strict performance requirements, such as DSP and real-time systems.

   The choice between Von Neumann and Harvard architectures depends on the specific computing requirements of the application, balancing factors like performance, cost, complexity, and power consumption.
