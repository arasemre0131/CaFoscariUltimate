PART B: SHORT ANSWERS

1. Injective (one-to-one): A function f: A → B is injective if for any distinct a1, a2 ∈ A, f(a1) ≠ f(a2). Example: f(x) = x^2 is injective on the set of positive real numbers.
Surjective (onto): A function f: A → B is surjective if for every b ∈ B, there exists an a ∈ A such that f(a) = b. Example: f(x) = x + 1 is surjective on the set of real numbers.

2. A recurrence relation is an equation that defines a sequence where each term is expressed in terms of the preceding terms. They are useful for solving problems that involve recursive structures, such as the Fibonacci sequence or the Tower of Hanoi problem.

3. Proof by induction:
Base case: For n = 1, Σ(i) = 1 = 1(1+1)/2
Inductive step: Assume the formula holds for n = k, i.e., Σ(i) = k(k+1)/2
For n = k+1, Σ(i) = Σ(i for i=1 to k) + (k+1) = k(k+1)/2 + (k+1) = (k+1)((k+1)+1)/2

4. The properties of an equivalence relation are:
- Reflexivity: (x, x) ∈ R for all x in the domain
- Symmetry: (x, y) ∈ R implies (y, x) ∈ R
- Transitivity: (x, y) ∈ R and (y, z) ∈ R implies (x, z) ∈ R
These properties allow you to partition a set into equivalence classes, where each element in the same class is related to every other element in that class.

5. A directed graph (digraph) is a graph where the edges have a specific direction, while an undirected graph has edges without any directional information.
Example of a directed graph: A road network where each road has a designated direction of travel.
Example of an undirected graph: A social network where the connections between individuals are not directed.

6. Using the Euclidean algorithm:
36 = 24 * 1 + 12
24 = 12 * 2 + 0
Therefore, gcd(36, 24) = 12.

PART C: ESSAY QUESTIONS

1. Discrete structures are fundamental to computer science as they provide the mathematical foundations for various data structures, algorithms, and computational models. Sets, relations, and graphs are essential for the design and analysis of algorithms:
- Sets are used to represent collections of objects and support operations like union, intersection, and complement, which are crucial for algorithm design.
- Relations describe connections between elements and are used to model various real-world problems, such as social networks, database schemas, and language grammars.
- Graphs are a powerful data structure for representing and analyzing networks, and are widely used in areas like routing, scheduling, and web page ranking.
Understanding these discrete structures allows computer scientists to develop efficient and effective algorithms to solve complex problems.

2. Direct proof: Directly shows that a statement is true by using logical reasoning and established facts.
Example: Proving that the sum of two even integers is even.
Indirect proof (proof by contradiction): Assumes the negation of the statement and shows that this leads to a contradiction, thereby proving the original statement.
Example: Proving that the square root of 2 is irrational.
Inductive proof: Establishes the truth of a statement for a base case and then shows that if the statement is true for a certain case, it is also true for the next case.
Example: Proving the formula for the sum of the first n positive integers.

The strengths of direct proof are its straightforward nature and the ability to provide a clear understanding of why a statement is true. Indirect proof is useful when the direct approach is not immediately obvious, and it can sometimes lead to deeper insights. Inductive proof is well-suited for problems involving recursive or iterative structures, but it requires identifying the appropriate base case and inductive step.

3. Example problem: Modeling a transportation network using a graph.
Representation: Vertices represent cities or towns, and edges represent the roads or transportation links between them. The edges can be weighted to represent the distance, travel time, or cost of the connection.
Graph properties:
- Connectivity: Determine if the network is connected, i.e., if there is a path between any two cities.
- Shortest path: Find the shortest path between two cities using algorithms like Dijkstra's algorithm or Floyd-Warshall algorithm.
- Minimum spanning tree: Identify the minimum-cost set of edges that connect all vertices, which can be used for efficient route planning.
- Centrality: Analyze the importance of cities in the network using measures like degree, betweenness, or closeness centrality.

By modeling the transportation network as a graph, we can leverage various graph algorithms and concepts to solve problems such as route optimization, network analysis, and infrastructure planning.
